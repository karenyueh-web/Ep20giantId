## 1. 📋 Order（一般訂單 — 聚合根）

**出現在**：訂單管理主頁 / 訂單明細 / CSV匯出入 / 批次操作  
**核心檔案**：[AdvancedOrderTable.tsx](file:///c:/Users/G00106917/Desktop/Ep2-giantId/src/app/components/AdvancedOrderTable.tsx)、[OrderStoreContext.tsx](file:///c:/Users/G00106917/Desktop/Ep2-giantId/src/app/components/OrderStoreContext.tsx)

### 1.1 識別與分類欄位

| 欄位名（英） | 欄位名（中） | 型別 | 必填 | 來源說明 |
|---|---|---|:---:|---|
| **🔑 id** | 系統識別碼 | number | ✓ | 內部唯一 ID |
| **orderNo** | 訂單號碼 | string | ✓ | SAP 訂單號，如 `400649723` |
| orderDate | 訂單日期 | string | ✓ | 格式 `YYYY/MM/DD` |
| orderType | 訂單類型 | string | ✓ | `ZOR / ZNB / Z2QB / Z2HB / Z1JB / Z1JD / Z3TB / Z3TD` |
| orderSeq | 訂單序號 | string | ✓ | 如 `10 / 20`，同單多行的序號 |
| docSeqNo | 單號序號 | string | ✓ | 組合欄位：`orderNo + orderSeq` |

### 1.2 物料欄位（→ 引用 Material 主檔）

| 欄位名（英） | 欄位名（中） | 型別 | 必填 | 來源說明 |
|---|---|---|:---:|---|
| **🔗 materialNo** | 料號 | string | ✓ | FK → Material，如 `1129-CSL0075-L01` |
| productName | 品名 | string | ✓ | 物料名稱 |
| specification | 規格 | string | ✓ | 規格描述 |
| vendorMaterialNo | 廠商料號 | string | △ | 廠商自有料號 |
| customerBrand | 客戶品牌 | string | △ | 如 `G01` |

### 1.3 數量欄位（OrderQty 值物件）

| 欄位名（英） | 欄位名（中） | 型別 | 必填 | 來源說明 |
|---|---|---|:---:|---|
| orderQty | 訂貨量 | number | ✓ | 訂單總數量 |
| acceptQty | 驗收量 | number | ✓ | 已完成驗收數量 |
| inTransitQty | 在途量 | number | ✓ | 已出貨未到廠 |
| undeliveredQty | 未交量 | number | ✓ | **計算欄位**：`max(0, orderQty − acceptQty − inTransitQty)` |
| deliveryQty | 交貨量 | number | △ | 已排程交貨量 |

### 1.4 價格欄位（OrderPricing 值物件）

| 欄位名（英） | 欄位名（中） | 型別 | 必填 | 來源說明 |
|---|---|---|:---:|---|
| comparePrice | 比對單價 | string | △ | 如 `33.34(有限定廠商)` |
| unit | 單位 | string | △ | `SET / PC / EA / PCS / KGM…` |
| **🔗 currency** | 幣別 | string | △ | FK → Currency，如 `TWD / USD` |
| leadtime | 前置時間 | number | △ | 交貨前置天數 |

### 1.5 交貨排程欄位（DeliverySchedule 值物件）

| 欄位名（英） | 欄位名（中） | 型別 | 必填 | 來源說明 |
|---|---|---|:---:|---|
| expectedDelivery | 預計交期 | string | ✓ | 巨大期望交貨日 `YYYY/MM/DD` |
| vendorDeliveryDate | 廠商可交貨日期 | string | △ | 廠商回覆的交貨日（**可編輯**） |
| agreedDate | 協議交期 | string | △ | 雙方協議確認的交貨日 |
| productionScheduleDate | 生管端交貨日期 | string | △ | 生管排程系統的交期 |
| scheduleLines | 排程行明細 | ScheduleLine[] | △ | 拆期/拆單後的多行排程 |
| **🔗 storageLocationCode** | 儲存地點代碼 | string | △ | FK → StorageLocation |

### 1.6 組織欄位（引用共用主檔）

| 欄位名（英） | 欄位名（中） | 型別 | 必填 | 來源說明 |
|---|---|---|:---:|---|
| **🔗 vendorCode** | 廠商編號 | string | ✓ | FK → Vendor |
| vendorName | 廠商簡稱(編號) | string | ✓ | 如 `速聯國際(00010046)` |
| **🔗 plantCode** | 工廠代號 | string | △ | FK → Plant |
| **🔗 purchaseOrg** | 採購組織 | string | △ | FK → PurchaseOrg |
| **🔗 companyCode / company** | 公司 | string | △ | FK → Company |
| purchaser | 採購人員 | string | △ | 負責採購人員姓名 |

### 1.7 備註欄位（OrderNotes 值物件）

| 欄位名（英） | 欄位名（中） | 型別 | 必填 | 來源說明 |
|---|---|---|:---:|---|
| lineItemNote | 單項小記 | string | △ | 對外可見備註 |
| internalNote | 項目註記(內部) | string | △ | 內部備註，**不顯示給廠商** |
| materialPOContent | 物料PO內文 | string | △ | SAP 訂購單文字內容 |
| gbdOrderNo | GBD單號 | string | △ | 集團內部 GBD 訂單號 |

### 1.8 狀態與標記欄位（OrderFlags 值物件）

| 欄位名（英） | 欄位名（中） | 型別 | 必填 | 來源說明 |
|---|---|---|:---:|---|
| status | 訂單狀態 | enum | ✓ | `NP / V / B / CK / CL`（見下方狀態流） |
| isRejectedOrder | 不接單標記 | boolean | △ | 廠商拒絕此訂單 |
| adjustmentType | 調整類型 | enum | △ | `modify / reject / split / split-order` |
| deletionCode | 刪除碼 | string | △ | 刪單修正單完成時寫入修正單號 |
| statisticalDeliveryDate | 統計交期 | string | △ | 統計用交期欄位 |

### 1.9 歷程與附件（引用共用 HistoryEntry）

| 欄位名（英） | 欄位名（中） | 型別 | 必填 | 來源說明 |
|---|---|---|:---:|---|
| history | 操作歷程 | HistoryEntry[] | △ | 系統/管理員操作紀錄 |
| replyHistory | 回覆歷程 | HistoryEntry[] | △ | 廠商回覆紀錄 |
| attachments | 附件 | FileAttachment[] | △ | 上傳的檔案列表 |

### 1.10 UI 控制旗標（僅前端使用）

| 欄位名（英） | 欄位名（中） | 型別 | 說明 |
|---|---|---|---|
| isNew | 新增標記 | boolean | 標記未讀新訂單 |
| isModified | 已修改標記 | boolean | 標記已修改未存 |
| isReturned | 已退回標記 | boolean | 標記已退回 |
| isRead | 已讀標記 | boolean | 標記已讀 |
| isSplit | 拆單標記 | boolean | 標記已拆單 |
| splitOrderSeq | 拆單序號 | string | 拆單後的新序號 |

**訂單狀態流程：**

```
NP (未處理) → V (廠商確認中) → B (採購確認中) → CK (訂單已確認) → CL (關閉結案)
                                                     ↑ 退回
                                          NP ←───────┘
```

| 狀態代碼 | 中文名稱 | 顏色 |
|:---:|---|---|
| `NP` | 未處理 | 🔴 紅色 `#b71d18` |
| `V` | 廠商確認中 | 🔵 青色 `#006c9c` |
| `B` | 採購確認中 | 🟣 紫色 `#5119b7` |
| `CK` | 訂單已確認 | 🟢 綠色 `#118d57` |
| `CL` | 關閉結案 | ⚪ 灰色 `#637381` |

## ⚠️ 注意事項與欄位命名不一致問題

| 問題 | 說明 | 建議 |
|------|------|------|
| 🔴 `company` vs `companyCode` | OrderRow 用 `company`，其他模組用 `companyCode` | 統一為 `companyCode` |
| 🔴 `plant` vs `plantCode` | SampleOrder 用 `plant`，OrderRow 用 `plantCode` | 統一為 `plantCode` |
| 🔴 `material` vs `materialNo` | SampleOrder 用 `material`，OrderRow 用 `materialNo` | 統一為 `materialNo` |
| 🟡 `vendor` vs `vendorCode + vendorName` | ForecastOrder 用合併 `vendor`，OrderRow 分開 | 建議分開為 `vendorCode + vendorName` |
| 🟡 `longSpec` vs `longDescription` vs `specification` | 排程查詢用 `longSpec`，索樣單用 `longDescription`，訂單用 `specification` | 統一長規格欄位名稱 |
| 🟡 `orderQty` 型別不一致 | OrderRow 用 `number`，CSV 解析用 `string` | 統一為 `number`，CSV 解析時轉型 |
| 🟡 `leadtime` vs `leadTime` | OrderRow 用 `leadtime`，ForecastOrder 用 `leadTime` | 統一為 `leadTime` |
