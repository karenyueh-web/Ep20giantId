# 🟡 跨模組共用主檔 — 9 個 Reference Objects 欄位完整定義

> **說明**：每個主檔分為五層：
> 🔑 識別鍵 ／ 📋 基本描述 ／ 💼 業務屬性 ／ 🔗 關聯鍵 ／ 🕐 稽核欄位

---

## 1. 🏭 Vendor（廠商主檔）

**出現在**：訂單 / 出貨 / 發票 / 品質異常 / 零件 / 廠商帳號  
**現況**：目前各模組只帶入 `vendorCode + vendorName`，缺完整主檔

| 欄位名（英） | 欄位名（中） | 型別 | 必填 | 來源依據 |
|---|---|---|:---:|---|
| **🔑 vendorCode** | 廠商代號（SAP） | string | ✓ | 所有模組共用鍵 `000100463` |
| vendorName | 廠商簡稱 | string | ✓ | 所有模組顯示用 |
| vendorFullName | 廠商全名 | string | ✓ | VendorAccountManagementPage |
| vendorType | 廠商類型 | enum | ✓ | `一般供應商 / 保稅廠商` |
| taxId | 統一編號 | string | △ | 發票開立需知 |
| phone | 聯絡電話 | string | △ | VendorData.phone |
| address | 地址 | string | △ | VendorData.address |
| countryCode | 廠商所在國 | string | △ | 出口報關用 → FK: Country |
| mainProducts | 主要產品描述 | string | △ | VendorData.mainProducts |
| status | 啟用狀態 | enum | ✓ | `active / inactive` |
| **🕐 createdAt** | 建立時間 | string | ✓ | 稽核 |
| **🕐 updatedAt** | 最後更新時間 | string | ✓ | 稽核 |

## 2. 🔩 Material（物料主檔）

**出現在**：訂單 / 出貨 / 發票 / 品質異常 / 零件  
**現況**：各模組帶入 `materialNo + productName + specification`，但欄位名稱不統一（specification / longDescription / description）

| 欄位名（英） | 欄位名（中） | 型別 | 必填 | 來源依據 |
|---|---|---|:---:|---|
| **🔑 materialNo** | 料號（SAP） | string | ✓ | 如 `1129-CSL0075-L01` |
| productName | 品名 | string | ✓ | OrderRow.productName |
| specification | 規格描述 | string | ✓ | OrderRow.specification |
| longDescription | 長規格敘述 | string | △ | PartRecord.longDescription（同義字） |
| materialType | 物料類型 | string | △ | SAP 物料類型碼 |
| unit | 基本計量單位 | string | △ | OrderRow.unit（PCE/SET/KGM…） |
| customerBrand | 客戶品牌 | string | △ | OrderRow.customerBrand |
| vendorMaterialNo | 廠商料號 | string | △ | OrderRow.vendorMaterialNo / PartRecord.vendorPartNo |
| **🔗 plantCode** | 預設工廠 | string | △ | FK → Plant |
| **🔗 purchaseOrg** | 採購組織 | string | △ | FK → PurchaseOrg |
| status | 啟用狀態 | enum | ✓ | `active / obsolete` |

> **⚠️ 注意**：`grossWeight / netWeight / weightUnit` 屬於 **WeightSpec（值物件 #9）**，不放在 Material 主檔中

## 8. 📜 HistoryEntry（操作歷程 — 共用 Value Object）

**出現在**：訂單 / 修正單 / 發票 / 零件  
**現況**：各模組欄位名稱略有差異，需統一

| 欄位名（英） | 欄位名（中） | 型別 | 必填 | 來源/差異說明 |
|---|---|---|:---:|---|
| **🔑 id** | 歷程識別碼 | string | ✓ | 若升為 Entity 需要（目前無） |
| entityType | 所屬物件類型 | enum | ✓ | `order / invoice / shipment / part`（目前無，各自存） |
| entityId | 所屬物件 ID | string | ✓ | 對應各主物件 ID（目前無） |
| timestamp | 操作時間 | string | ✓ | 訂單用 `date`；發票用 `timestamp` → **需統一** |
| event | 事件/操作類型描述 | string | ✓ | 訂單用 `event`；發票用 `action` → **需統一** |
| operator | 操作人員 | string | ✓ | 所有模組一致 |
| remark | 備註/變更摘要 | string | △ | 訂單用 `remark`；發票用 `changes` → **需統一** |

> ⚠️ **當前問題**：`HistoryEntry` 在 `OrderStoreContext.tsx` 與 `invoiceStore.ts` 有兩個不同定義，欄位名稱不一致，實際串接 API 時需要統一。

**建議統一介面：**
```typescript
interface HistoryEntry {
  id:         string;   // UUID
  entityType: 'order' | 'invoice' | 'shipment' | 'part' | 'correction';
  entityId:   string;
  timestamp:  string;   // YYYY/MM/DD HH:mm:ss
  event:      string;   // 統一用 event
  operator:   string;
  remark:     string;   // 統一用 remark
}
```
